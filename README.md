INSTRUCTIONS:

Before running the executable of Yagbu, first move the file you want to be encrypted/decrypted to the same directory with the executable.

When you run the executable, you will be greeted with a CLI menu.

After you choose either encryption or decryption; the program will ask you for a file name, a seed and a key.

For example, if you had moved a file named "The File.exe" to Yagbu's directory to encrypt, you need to write the full name of the file
"The File.exe" (without the quotation marks ofc)

If you had moved an already encrypted file named "The File.yagbu" to Yagbu's directory to decrypt, again you need to write
the full name of the file, in this case "The File.yagbu" (again without the quotation marks)

After a successful decryption, the fill name will be "The File" without any file type association in this example above. 
So, you need to remember the original file type and add it yourself via renaming the file.
(I decided to make this change purely because of to hide the original file type for a better sense of privacy.)

As for the seed and the key, they both have to be integers. While the seed can be both positive and negative integers, 
the key HAS TO BE a positive integer.

The sizes of the numbers aren't a problem AS LONG AS YOU DON'T FORGET what you chose for your seed and key. 
(Obviously don't enter a hundred gazillion, as python has its limits.)

After you enter the seed and key you chose, the program will encrypt/decrypt your file.

When decrypting, enter the seed and the key you had chosen for encrypting and the program will decrypt your file.

!!! ALWAYS CHECK THE SEED AND KEY YOU CHOSE BEFORE PROCEEDING, AS ANY TINY MISTAKE WILL BE CATASTROPHIC !!!
